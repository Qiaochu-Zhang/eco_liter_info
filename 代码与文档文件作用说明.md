# 代码与文档文件作用说明

## 1. 项目概览

本项目用于跟踪 6 本经济学期刊的新文章，完成采集、去重、规则分类，并导出为 Excel 文件。当前仓库的实际实现以 `main.py` 为入口，核心代码集中在 `economics_tracker/` 目录，测试位于 `tests/`，示例输出位于 `output/`。

当前主流程是：

1. 读取期刊配置
2. 从 RSS 抓取文章，必要时用 Crossref 兜底
3. 富化作者、摘要、DOI、日期、JEL 等字段
4. 过滤无关标题并去重
5. 按 JEL 和关键词规则分类
6. 导出为 Excel

## 3. `economics_tracker/` 代码目录说明

### `economics_tracker/__init__.py`

包初始化文件，当前只用于标识这是一个 Python package，没有业务逻辑。

### `economics_tracker/config.py`

全局配置中心。

主要内容：

- `JournalConfig`：期刊配置数据结构
- `JOURNALS`：当前追踪的 6 本期刊及其主页、RSS、ISSN
- `PRIORITY_JEL_PREFIXES`：优先保留 JEL 前缀
- `DOWNWEIGHT_JEL_PREFIXES`：降权但保留复核的 JEL 前缀
- `REJECT_JEL_PREFIXES`：默认排除的 JEL 前缀
- `HIGH_VALUE_KEYWORDS`：高价值关键词，命中可直接提升为 `selected`
- `WEAK_SIGNAL_KEYWORDS`：弱信号关键词，通常会进入 `review`
- `EXPORT_COLUMNS`：Excel 导出字段顺序

它相当于整个项目的“规则表”和“期刊列表”。

### `economics_tracker/models.py`

统一数据模型定义。

核心数据结构：

- `Article`

主要字段：

- `journal`
- `title`
- `url`
- `date`
- `authors`
- `abstract`
- `doi`
- `jel_codes`
- `matched_keywords`
- `decision`
- `reason`
- `source`

主要方法：

- `dedupe_key()`：生成去重键，优先 DOI，其次 URL，最后是 `journal + title`
- `to_row()`：把文章对象转成适合导出到 Excel 的字典

### `economics_tracker/classifier.py`

文章分类模块。

核心职责：

- 根据标题和摘要匹配关键词
- 根据 JEL 前缀判断优先级
- 给每篇文章生成分类结果

关键结构：

- `ClassificationResult`：保存 `decision`、`reason`、`matched_keywords`

分类逻辑概括：

- 命中高价值关键词：`selected`
- 命中优先 JEL：`selected`
- 命中降权 JEL：`review`
- 仅命中弱关键词：`review`
- 命中排除 JEL 且没有 override：`rejected`
- 没有任何有效信号：`rejected`

### `economics_tracker/exporter.py`

Excel 导出模块。

核心职责：

- 创建 Excel 工作簿
- 建立 `selected` 和 `rejected` 两个 sheet
- 按 `EXPORT_COLUMNS` 固定顺序输出字段

当前导出行为：

- `decision == "selected"` 写入 `selected`
- 其他结果，包括 `review` 和 `rejected`，都写入 `rejected`

### `economics_tracker/merger.py`

Excel 合并模块。

核心职责：

- 读取多个已有的 xlsx 输出文件
- 从各 sheet 中恢复为 `Article` 列表
- 再次按项目去重规则合并
- 重新导出为一个新的 Excel 文件

适用场景：

- 合并多天结果
- 合并测试输出
- 生成周报或月报型汇总文件

### `economics_tracker/pipeline.py`

主流程编排模块。

核心函数：

- `filter_junk()`：过滤标题中包含 `front matter`、`recent referees` 的记录
- `dedupe_articles()`：对采集结果按统一规则去重
- `apply_classification()`：为每篇文章调用分类器并回填结果
- `run_pipeline()`：将采集、过滤、去重、分类、导出串成完整流程

这是项目最核心的 orchestration 文件。

### `economics_tracker/sources.py`

数据采集与富化模块，也是仓库里逻辑最复杂的文件。

主要职责分成几层：

#### 1. 网络抓取基础能力

- `fetch_url()`：统一封装 HTTP 请求
- 设置默认超时和 User-Agent

#### 2. RSS / Atom 解析

- `_MetaParser`：解析文章页面里的 `meta` 和 `JSON-LD`
- `parse_rss_feed()`：从 RSS / Atom 中抽取标题、链接、日期、摘要、作者、DOI
- `_extract_doi_from_item()`：从 RSS 常见字段里提取 DOI
- `_find_text()`、`_find_link()`、`_split_authors()`、`clean_html()`：做字段提取和清洗

#### 3. 文章页面富化

- `enrich_from_journal_page()`：访问文章详情页，通过 `meta`、`JSON-LD` 和正文补充摘要、DOI、日期、作者、JEL
- `_extract_jel_codes()`：从页面文本中提取 JEL 形式编码

#### 4. Crossref 富化与回源

- `enrich_from_crossref()`：在文章已有 DOI 时，从 Crossref 拉取结构化元数据
- `collect_from_crossref_journal()`：当 RSS 为空时，用期刊 ISSN 和日期范围从 Crossref 拉取期刊文章

#### 5. 第三方 provider 抽象

- `RepecIdeasProvider`：当前只是预留接口，`enrich()` 还没有真实实现

#### 6. 主采集器

- `JournalSourceCollector`：遍历 `JOURNALS`，逐刊执行采集和富化

当前单期刊采集顺序是：

1. 先抓 RSS
2. 如果 RSS 失败或为空，则按“当月”范围调用 Crossref journal fallback
3. 对每篇文章尝试 provider 富化
4. 对非 `crossref-journal` 来源的文章再做 Crossref DOI 富化
5. 最后尝试文章页富化

补充说明：

- 当前 `collect()` 没有日期参数，Crossref fallback 使用的是“当前月 1 日到今天”
- `RepecIdeasProvider` 目前是占位实现，因此当前有效数据源主要还是 RSS、Crossref 和文章页

## 4. `tests/` 测试文件说明

### `tests/test_classifier.py`

分类逻辑测试。

覆盖场景：

- 优先 JEL 是否进入 `selected`
- 高价值关键词是否能覆盖默认排除 JEL
- 降权 JEL 是否进入 `review`
- 无有效信号是否进入 `rejected`

### `tests/test_exporter.py`

Excel 导出测试。

覆盖场景：

- `selected` 是否进入 `selected` sheet
- `rejected` 是否进入 `rejected` sheet
- 基本写表行为是否正确

说明：

- 该测试依赖 `openpyxl`
- 如果未安装，测试会被跳过

## 6. GitHub Actions 与自动化文件

### `.github/workflows/weekly_merge.yml`

自动执行工作流。

虽然文件名叫 `weekly_merge.yml`，但当前实际配置是“每月任务”：

- `name: Monthly Journal Fetch`
- 定时表达式为每月 10 日 UTC 05:00 执行

具体动作：

1. checkout 仓库
2. 安装 Python 3.11
3. 安装依赖
4. 计算当月标签 `YYYY-MM`
5. 运行 `python main.py --output output/monthly/<YYYY-MM>.xlsx`
6. 若有变更则提交并推送输出文件

说明：

- 文件名和工作流用途之间存在命名不一致
- 从实际内容看，它更像月度抓取任务，而不是周合并任务

## 9. 当前仓库的结构关系总结

如果按“职责链”来看，当前项目可以理解为：

- `task.md`、`IMPLEMENTATION_PLAN.md`、`项目流程说明.md`：定义需求、方案和流程背景
- `main.py`：命令行入口
- `economics_tracker/sources.py`：采集和富化
- `economics_tracker/pipeline.py`：流程编排
- `economics_tracker/classifier.py`：分类判断
- `economics_tracker/exporter.py`：Excel 输出
- `economics_tracker/merger.py`：多文件汇总
- `tests/`：验证核心分类和导出逻辑
- `output/`：保存运行结果样例
- `.github/workflows/weekly_merge.yml`：定时自动运行

也就是说，这个仓库已经具备一个可运行的“经济学期刊自动抓取与筛选导出”最小闭环，实现上以规则驱动为主，第三方元数据富化以 Crossref 为当前主要可用来源，RePEc / IDEAS 仍处于接口预留状态。
